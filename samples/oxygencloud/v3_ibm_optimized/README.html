Integrated Video API Documentation

This document is a brief introduction to Integrated Video and how to use the API on your site.

Dependent Files

The iv folder contains a few files that will need to be included in your page.

    Stylesheets

  • Integrated Video stylesheet:iv/css/iv.css
  • Video JS stylesheet:iv/css/video-js.css
  • Video JS quicktime-skin:iv/css/vjs-quicktime-skin.css

    • Scripts

  • jQuery library:iv/js/jquery-1.9.0.min.js
  • Video JS library:iv/js/video.dev.js
  • Integrated Video core:iv/js/iv.core.js
  • Integrated Video mobile:iv/js/iv.mobile.basic.js
  • Integrated Video API:iv/js/iv.html5.js

Make sure to include jQuery and Video JS before the Integrated Video API as it is dependent on them.

Markup Structure

An Integrated Video has three basic components:

  • Initial viewable content
  • A video that is not visible on page load
  • A container which is an interactive area that controls the visibility of the video and the content

Initial Viewable Content

This is a group of elements that occupy the same space as the video. It could be text, images, even another video. Pretty much any content that you want to put on top of the video goes here. It all needs to be within the same common parent node with id="iv-container". The img tag with id="iv-poster" is used as an image poster, when the video cannot be preloaded. Some mobile devices like iPhones and iPads will NOT preload the video in order to protect their users' bandwidth, in that case the image is used as poster.

The Video

The video is included as a native video. The video tag must have an id. You should also give it a class of integrated-video.

The Container

The container can be any element that is a parent of both the video and the content. This will act as the interactive area by default. It should have an id and a class of integrated-video-container.

Primary API

All integrated Video functionality is accessed through the global object IV. IV.Element is the primary constructor function.

IV.Element Initialization

The IV.Element is initialized at the end of the integrate_video.html5.js file, after all its methods have been defined. The IV.Element requires one argument passed to it, an object literal. The argument requires 3 properties, container, content, and video. These properties can either be a string with the id of it's respective dom element, or it can be the dom element itself.
